## Best Practice Examples

### Model Course Structure (from docker_complete_enhanced/lesson-1.yml)

```yaml
exercises:
  - type: terminal
    slug: lesson-1-term
    sequence_order: 1
    description: Check CI/CD workflow status
    command: git log --oneline -5
    validation:
      must_not_include:
        - Error
        - fatal
    hints:
      - View recent commits to see CI/CD activity
    timeout_sec: 60
    require_pass: true

  - type: mcq
    slug: lesson-1-mcq
    sequence_order: 2
    question: What is the main difference between...?
    options:
      - Correct answer
      - Wrong answer
      - Wrong answer
      - Wrong answer
    correct_answer_index: 0
    explanation: Detailed explanation...
    require_pass: true

  - type: reflection
    slug: lesson-1-reflect
    sequence_order: 3
    prompt: What are the trade-offs...?

  - type: checkpoint
    slug: lesson-1-checkpoint
    sequence_order: 4
    prompt: Describe the stages of...
```

### Key Elements for Command-Based Courses:

1. **Terminal Exercise**: Hands-on command practice
   - Clear description
   - Realistic command
   - Validation rules
   - Helpful hints
   - Timeout protection

2. **MCQ**: Test conceptual understanding
   - Clear question
   - 4 options
   - Explanation for correct answer

3. **Reflection**: Critical thinking
   - Open-ended prompt
   - Real-world application focus

4. **Checkpoint**: Knowledge verification
   - Requires explanation
   - Tests comprehensive understanding

- ‚úÖ 1 terminal exercise per lesson minimum

**Example Terminal Exercise Progression:**

